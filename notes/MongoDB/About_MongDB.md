# MongoDB Core Concepts

## What is MongoDB

MongoDB is a **NoSQL database**.
It is very **flexible** compared to traditional SQL databases.

MongoDB stores data in a **JSON-like format called documents**.

However, internally MongoDB stores data in **BSON (Binary JSON)**, which is a binary version of JSON that is faster and supports more data types.

Users see and work with data in **JSON document format**, but the database engine stores it as **BSON**.

---

# SQL vs NoSQL

## SQL

SQL stands for **Structured Query Language**.

In SQL databases:

* Data is stored in **tables**
* Tables contain **rows and columns**
* The structure (schema) is fixed

Example:

School Database

Tables inside it may include:

* Student table
* Teacher table
* Course table

Each table has rows and columns.

---

## NoSQL (MongoDB)

MongoDB does not store data in tables.

Instead it stores data in:

* **Documents**
* Inside **Collections**
* Inside **Databases**

MongoDB is flexible because documents inside a collection can have different structures.

---

# Database

A **database** is the main container where data is stored.

Example:

```
schoolDB
```

Inside one database there can be multiple collections.

---

# Collection

A **collection** is similar to a **table in SQL**.

It stores multiple documents.

Example:

```
schoolDB
   └── students
   └── teachers
   └── users
```

When we run the command:

```
show collections
```

MongoDB shows the collections present in the current database.

Example output:

```
users
students
```

---

# Document

A **document** is a single record stored inside a collection.

Documents are stored in **JSON-like format**.

Example:

```
{
  "name": "Mitali",
  "age": 20,
  "city": "Pune"
}
```

Multiple documents exist inside a collection.

---

# _id Field

Every document in MongoDB contains a special field called:

```
_id
```

The `_id` field:

* Acts like a **Primary Key**
* Is **unique for every document**
* Is **automatically generated by MongoDB** if not provided

Example document:

```
{
  "_id": ObjectId("65a9d1..."),
  "name": "Mitali",
  "age": 20
}
```

---

# MongoDB Data Structure

The structure of MongoDB is:

```
Database
   → Collection
        → Document
```

Example:

```
schoolDB
   → students
        → {name: "Mitali", age: 20}
        → {name: "Riya", age: 21}
```

* **Database** contains collections
* **Collections** contain documents
* **Documents** store the actual data
